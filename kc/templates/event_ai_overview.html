{# ── Model description ── #} {% if model_name == "model1" %}
Model 1 — Derivative-peak detection

Candidate generation: Scans the signal's first derivative and picks time-points where the rate of change peaks sharply. These derivative peaks indicate rapid transitions — the start of aggregation, a secondary rise, or a sudden drop. A set of broad "trend" windows (baseline, mid-transition, plateau) is also added to catch slow events.

Scoring: Each candidate is scored by a RandomForest classifier trained on submitted data (binary: event / no-event). Without enough training data the heuristic fallback weights derivative-peak count (40%), residual from a global sigmoid (30%), local slope (15%) and curvature (15%).

Event typing: After a location is confirmed, the box's temporal position in the curve (baseline / transition / plateau) is combined with local signal characteristics to assign a specific type — e.g. a box in the baseline with positive slope → Creeping baseline; a box near the end with the signal still rising → Not finished aggregation.

Strength: Good at sharp, well-defined events. Weakness: May miss slow monotonic drift that has no derivative spike.

{% elif model_name == "claude" %}
Claude's Model — Residual & drift detection

Candidate generation: Instead of looking for derivative spikes, this model first fits a smooth 4-parameter logistic (4PL) sigmoid to the whole curve. It then generates dense sliding windows at three time-scales (12 %, 22 %, 35 % of the total duration) across the entire curve, plus extra windows centred on the highest residual peaks (where the real signal deviates most from the ideal sigmoid).

Features: Each window is described by residual statistics (mean, std, skew), lag-1 autocorrelation of the residuals (high autocorrelation = persistent drift), multi-scale derivative ratio (fine vs coarse — separates spikes from trends), an irregularity index, and temporal position. None of these features overlap with Model 1.

Scoring: A GradientBoosting classifier is trained on submissions that include Claude features (newer submissions). The heuristic fallback weights residual deviation (35%), residual noise (25%), autocorrelation/drift (25%) and irregularity (15%).

Event typing: Shared domain-knowledge rules with Model 1 (same classify_box_event_type function). Overlap between adjacent candidates is rejected at a stricter 40 % threshold (vs 50 % in Model 1), so events placed close together are more likely to be kept as distinct.

Strength: Designed to find slow-drift events (creeping baseline, only drift, laggy plateau) that Model 1 misses. Weakness: Dense candidate windows may produce false positives in very noisy curves until enough training data exists.
